# Bitratata — записи Битрикс24: загрузка, расшифровка, сегментация

Функционал как в [sa](https://github.com/tserashchuk/sa): скрипт получает на вход **количество записей** и **количество клиентов**. Сам выкачивает записи с диска Битрикс24, по одной расшифровывает (Vosk), складывает транскрипции в БД, отправляет их в DeepSeek и выводит результаты сегментации на главной странице.

**Цепочка:** загрузка → транскрибация (по одной, в БД) → DeepSeek → результаты на главной.

## Установка

**С виртуальным окружением (рекомендуется):**

```bash
cd bitratata
python -m venv .venv
.venv\Scripts\activate   # Windows
pip install -r requirements.txt
```

**Без venv** — установка в системный Python (один проект на машине):

```bash
cd bitratata
pip install -r requirements.txt
```

Дальше команды те же: `python manage.py runserver`, `celery -A config worker ...` — запускайте из этой же папки, без активации venv.

Скачайте [модель Vosk](https://alphacephei.com/vosk/models) (например, `vosk-model-small-ru-0.22`). 
Путь к модели и ключ DeepSeek настраиваются в `config/settings.py` (или через переменные окружения):

```python
VOSK_MODEL_PATH = os.environ.get(
    "VOSK_MODEL_PATH",
    str(BASE_DIR / "vosk-model-ru-0.42"),
)
DEEPSEEK_API_KEY = os.environ.get("DEEPSEEK_API_KEY", "")  # сюда ключ DeepSeek

# Для варианта с диском Битрикс24 (вместо телефонии):
BITRIX_STORAGE_NAME = os.environ.get("BITRIX_STORAGE_NAME", "Общий диск")
BITRIX_FOLDER_PATH = os.environ.get("BITRIX_FOLDER_PATH", "").strip()
```

## Запуск

```bash
python manage.py migrate
python manage.py createsuperuser
python manage.py runserver
```

Worker и Beat (нужен **Redis** на localhost:6379):

```bash
celery -A config worker -l info -B
```

Если Redis не установлен и появляется *Error 10061 connecting to localhost:6379*, можно запустить **тест без Redis**: задачи будут выполняться сразу в процессе (worker не нужен). В PowerShell перед запуском Django:

```powershell
$env:CELERY_EAGER="1"
python manage.py runserver
```

Либо установите и запустите Redis (например, через WSL или [Redis для Windows](https://github.com/microsoftarchive/redis/releases)).

## Личный кабинет

- **Регистрация** — `/accounts/register/`.
- **Главная** (`/`) — для пользователей: последние запуски и теги по контактам; ссылки на «Мои задания», «Мои клиенты».
- **Задания создаются из кабинета** — в разделе **Мои задания** (`/tasks/`) форма «Создать задание» (название, количество записей). **Запуск только вручную** (кнопка «Запустить»). Используется **общий ключ DeepSeek** (одна настройка задачи с `user` = пусто в админке).
- **Один вебхук на пользователя** — в разделе **Мой вебхук Битрикс24** (`/connection/`) пользователь указывает свой URL вебхука; при создании задания используется только он (количество подключений = 1).
- **Мои клиенты** (`/contacts/`) — пользователь видит **только своих** контактов (сегментированных тегами). По клику на контакт открывается страница с **тегами и транскрипциями** всех записей по этому контакту.
- **Лимит записей** — в админке в **Профили пользователей** задаётся «Лимит записей на запуск»; при создании и запуске задания проверяется.

Звонки группируются по контакту CRM (лид/контакт/сделка из телефонии Битрикс). Транскрипции и теги по контакту **обновляются** при повторном запуске.

## Настройки в админке

- **Подключения Битрикс24** — можно создавать из админки с привязкой к пользователю; обычно пользователь добавляет вебхук сам в кабинете (`/connection/`).
- **Задания Битрикс24** — создаются из кабинета; при необходимости их можно создавать и в админке (с указанием пользователя).
- **Профили пользователей** — общий лимит записей (пусто = без ограничения).

## Тестовый запуск

1. **Вариант А (с Redis):** запустите `celery -A config worker -l info -B`.  
   **Вариант Б (без Redis):** задайте переменную `CELERY_EAGER=1` и запустите только `python manage.py runserver` — задачи выполнятся в процессе запроса.
2. В админке откройте раздел **Задания Битрикс24** → выберите одно задание (или создайте новое: укажите записей 2–3, клиентов 1, выберите настройку задачи, сохраните).
3. Отметьте задание галочкой, в списке действий выберите **«Запустить сейчас (тестовый запуск)»** → **Выполнить**.
4. Задача уйдёт в очередь. Результат смотрите в логах worker, на главной странице (`/`) или в разделах **Запуски** / **Результаты сегментации** в админке.

## Главная страница

На главной (`/`) выводятся последние запуски и **результаты сегментации** — ответ DeepSeek по транскрипциям (выводы, темы, рекомендации).
